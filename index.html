            📜 LUA Library Information
          

          

        

        

          
          
          
          
        

      


      
      

        

          The Lua function library is shared by all Custom Exploit detections. It allows you to write Lua functions that can be referenced by JSON documents to implement complex data generation or parsing rules.


          All function names must start with qrdiuser_. Only function definitions are permitted — no global variables or code outside of functions.
        


        

          
📄

          
Click to browse and select a Lua library file

          
Accepted: .lua, .txt · Max size: 1 MB

        

        


        

          
          
        


        

          
          
          
Only one Lua library file is allowed per subscription. Saving a new file replaces the existing one.

        


        

          
⚡ Lua Runtime Constraints (Custom Exploit Validation)

          

            

              1 MB
              Max source file size
            

            

              4 MB
              Max RAM per detection
            

            

              100,000
              Max Lua opcodes per detection
            

            

              5.3
              Lua version
            

          

          

            ⚠ The opcode limit can burn out in milliseconds for heavy operations like cryptography. Keep functions lean.
          

          
Key CEV System Functions

          

            

              qrdisystem_unget(data)
              Throws unread bytes back into the memory buffer — safe for parsing continuous data streams without destroying the next message boundary.
            

            

              qrdiuser_*
              All your function names must start with this prefix. Global variables and code outside functions are not permitted.
